# Canon Rule #69: Trunk Only Serves What's in dist/

**Status:** ENFORCED


**Severity:** HIGH
**Version:** 1.0.0
**Date:** 2025-01-16

**Category:** build-tooling
**Tags:** trunk, dist, assets
**Language:** EN

---

**Intro:**
Trunk serves only files present in the dist directory, ignoring source folders like public or style. Misunderstanding this leads to missing assets at runtime.

**Problem:**
assets outside dist are not served causing 404 errors

**Solution:**
ensure all assets are built or copied into dist before serving

**Signals:**
- 404 styles.css
- public not served
- asset not found

**Search Intent:**
why trunk not serving files from public folder

**Keywords:**
trunk dist assets, leptos trunk css 404, trunk asset pipeline, dist only serving trunk

---

## Principle

Trunk serves **ONLY** the content that exists in the final `dist/` directory.

> **`public/`, `style/`, `assets/` are NOT served automatically.**  
> If it's not in `dist/`, it doesn't exist to the browser.

---

## The Problem

Developers assume Trunk works like Vite or Webpack Dev Server:
- "Put assets in `public/` and they're served"
- "CSS in `style/` is automatically available"
- "Trunk will find it"

**Wrong.** Trunk has a different model:

1. Processes `index.html` and its `data-trunk` directives
2. Copies/generates assets to `dist/`
3. Serves **only** `dist/`

**Result:** Assets in `public/styles.css` return 404 because they're not in `dist/`.

---

## Forbidden Assumptions
```bash
# ❌ WRONG ASSUMPTIONS
project/
├─ public/
│  └─ styles.css          # "Trunk will serve this" ← NO
├─ style/
│  └─ globals.css         # "Available at /style/globals.css" ← NO
├─ assets/
│  └─ logo.png            # "Browser can access this" ← NO
└─ dist/                  # Only this directory is served
```

### What Actually Happens
```bash
# Request
GET /styles.css

# Trunk looks for
dist/styles.css           # Not public/styles.css

# Result
404 Not Found
```

## Canonical Asset Flow

### 1. Source Files (NOT Served)
```
project/
├─ style/
│  └─ globals.css         # Source
├─ assets/
│  └─ logo.png            # Source
```

### 2. Build Process
```toml
# Trunk.toml
[[hooks]]
stage = "pre_build"
command = "npm"
command_arguments = ["run", "build:css"]
```
```json
// package.json
{
  "scripts": {
    "build:css": "tailwindcss -i style/globals.css -o dist/styles.css"
  }
}
```

### 3. Final Output (Served)
```
dist/
├─ styles-4aad46ee.css    # Generated by Trunk from dist/styles.css
├─ logo-a3f2b1c4.png      # Copied by Trunk if referenced
├─ index.html
└─ app-HASH.wasm
```

**Only files in `dist/` are accessible to browsers.**

## Trunk Directives Explained

### CSS Assets
```html
<!-- index.html -->
<link data-trunk rel="css" href="dist/styles.css" />
```

**What happens:**
1. Trunk reads `dist/styles.css` (must exist before Trunk runs)
2. Copies it to `dist/styles-HASH.css`
3. Updates HTML reference to hashed version
4. Browser requests `/styles-HASH.css` (exists in `dist/`)

### Static Assets
```html
<link data-trunk rel="copy-dir" href="public" />
```

**What happens:**
1. Trunk copies entire `public/` directory to `dist/`
2. `public/logo.png` becomes `dist/logo.png`
3. Browser can request `/logo.png`

## Common Mistakes

### Mistake 1: Generating to `public/`
```json
// ❌ WRONG
{
  "scripts": {
    "build:css": "tailwindcss -o public/styles.css"
  }
}
```

**Problem:** `public/styles.css` is never copied to `dist/`.

**Fix:**
```json
// ✅ CORRECT
{
  "scripts": {
    "build:css": "tailwindcss -o dist/styles.css"
  }
}
```

### Mistake 2: Assuming `style/` is Served
```html
<!-- ❌ WRONG -->
<link href="/style/globals.css" />
```

**Problem:** `style/` is a source directory, not served.

**Fix:**
```html
<!-- ✅ CORRECT -->
<link data-trunk rel="css" href="dist/styles.css" />
```

### Mistake 3: Direct References Without Trunk
```html
<!-- ❌ WRONG in Trunk projects -->
<link href="/styles.css" />
```

**Problem:** No cache busting, no copy to `dist/`.

**Fix:**
```html
<!-- ✅ CORRECT -->
<link data-trunk rel="css" href="dist/styles.css" />
```


### Check 2: File is Served


## Canonical Build Pipeline
```bash
# 1. Generate assets to dist/
npm run build:css
# Output: dist/styles.css

# 2. Trunk processes and serves
trunk serve --port 3001
# Reads: dist/styles.css
# Creates: dist/styles-HASH.css
# Serves: only dist/
```

## Architecture Diagram
```
Source Layer (NOT served)
├─ style/globals.css
├─ assets/logo.png
└─ public/

        ↓ Build Process
        
Build Layer (MUST exist before Trunk)
└─ dist/
   ├─ styles.css
   └─ logo.png

        ↓ Trunk Processing
        
Served Layer (Browser accessible)
└─ dist/
   ├─ styles-HASH.css
   ├─ logo-HASH.png
   ├─ index.html
   └─ app-HASH.wasm
```

---

## Canonical Justification

> **Trunk is a static asset bundler, not a development server.**  
> It transforms and serves `dist/`, nothing else.

This constraint ensures:
- **Deterministic builds** — output is always `dist/`
- **Cache busting** — assets get content hashes
- **Production parity** — dev and prod serve same structure
- **Clear separation** — source vs. output

Trying to serve `public/` or `style/` directly breaks this model.
